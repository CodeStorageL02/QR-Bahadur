This document serves as a detailed guide for project implementation. After decompression, please open the project "__QR-Profile__" directly in an integrated development environment (IDE). For the RR method, we invoked the R language, therefore, both the R compiler and the rpy2.robjects package in Python should be properly installed.

The entire project is divided into two parts: Case study and Simulation study. The Case study folder contains all the codes for the case presented in the paper.  
The Simulation study folder consists of seven subfolders, in which Determinate-K folder contains the codes for selecting K; Determinate-T folder contains the codes for selecting T; MR folder simulates all the OOC ARLs for the MR method; QR-BR folder simulates all the OOC ARLs for the QR-BR method; QR-LS folder simulates all the OOC ARLs for the QR-LS method; RR folder simulates all the OOC ARLs for the RR method; Table1-Time folder provides the codes for the computational time comparison.

In the __Implementation_guide.docx__ file, we provide, in tabular form, the code corresponding to the results presented in the paper to facilitate their reproduction. Additionally, __File_Tree.txt__ contains detailed annotations, in the form of a file tree, explaining the functionality of each code.
